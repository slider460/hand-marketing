# Развёртывание hand-marketing.ru

Готовый к публикации сайт лежит в папке **`mirror/`** — это статическая копия (HTML + CSS + JS + картинки), полностью автономная, без зависимости от Tilda. Серверная часть нужна только для приёма заявок с форм (PHP, файл `mirror/api/lead.php`).

## Что внутри `mirror/`
- 57 страниц сайта (главная, услуги, 45 кейсов, контакты и т.д.) + 2 новых кейса (`portfolio/samara-stand-vdnh`, `portfolio/samara-exhibition`)
- `api/*.json` — данные каталога проектов (отдаются локально, без обращений к Tilda)
- `api/lead.php` — обработчик форм (e-mail + запись в `leads.csv`)
- `.htaccess` — Apache: красивые URL, кэш, gzip, заголовки безопасности
- `static/` — вся графика, шрифты и скрипты локально

Внешних обращений к Tilda нет. Снаружи подгружаются только Яндекс.Метрика, Яндекс.Карты (на /contacts) и видео (Dropbox/YouTube).

## Вариант 1. Через GitHub (рекомендуется)

### 1. Создать репозиторий и запушить (выполнить один раз в терминале на этой машине)
```bash
cd /Users/aleksandrnarodetskii/Downloads/hand-marketing-react

# авторизоваться в GitHub (откроется браузер)
gh auth login

# создать приватный репозиторий и запушить
gh repo create hand-marketing-site --private --source=. --remote=origin --push
```
Если `gh` нет — создайте репозиторий вручную на github.com, затем:
```bash
git remote add origin https://github.com/<ваш-логин>/hand-marketing-site.git
git push -u origin main
```

### 2. Выложить на ваш хостинг
Зависит от хостинга. Два рабочих способа:

**A. Git pull на сервере** (если есть SSH и git):
```bash
ssh user@ваш-сервер
cd ~/                      # или туда, где лежит сайт
git clone https://github.com/<логин>/hand-marketing-site.git
# настроить веб-корень (DocumentRoot) на .../hand-marketing-site/mirror
```
Для обновлений потом: `git pull` на сервере.

**B. Загрузить содержимое `mirror/`** в `public_html` (или корень сайта) по FTP/панели хостинга.
> Важно: в `public_html` кладётся **содержимое** папки `mirror/` (файлы `index.html`, `static/`, `api/`, `.htaccess` …), а не сама папка `mirror`.

## Вариант 2. Без GitHub — просто загрузить файлы
Скопировать всё из `mirror/` в корень сайта на хостинге (FTP/файловый менеджер панели). Готово.

## После публикации — проверить
1. Открывается главная, кейсы, контакты — везде графика на месте.
2. Каталог «Проекты» показывает карточки (данные из `api/*.json`).
3. **Форма заявки:** укажите получателя в `mirror/api/lead.php` (строка `$TO`). Отправьте тестовую заявку — должно прийти письмо и появиться запись в `api/leads.csv`.
4. Если хостинг — Apache, убедитесь, что включены модули `mod_rewrite`, `mod_headers`, `mod_expires` (обычно включены).

## Требования хостинга
- Любой Apache/Nginx с поддержкой статики.
- PHP ≥ 7.4 — только для форм (`api/lead.php`). Без PHP сайт тоже работает, но заявки не будут отправляться.
