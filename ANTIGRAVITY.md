# ANTIGRAVITY.md — ТЗ для доработки и запуска hand-marketing.ru

Этот проект — миграция сайта hand-marketing.ru с Tilda на React/Vite.
Весь контент (58 страниц, 43 кейса, 358 изображений) автоматически извлечён
с живого сайта и лежит в структурированном виде. Сайт собирается и работает
из коробки; ниже — что нужно доделать перед запуском и как.

## Архитектура

- **Стек**: React 18 + TypeScript + Vite 5 + Tailwind 3 + react-router-dom 6.
  Тот же стек, что у weshow.su — проекты консолидированы.
- **Дизайн-система v2.2**: Montserrat (display) / Inter (body), полихромная
  палитра из изометрического куба-логотипа (6 семей × 3 оттенка) — см.
  `tailwind.config.js` и `src/styles/index.css`. Primary — зелёный `#96c223`.
- **Кейс-карточки (важно, фирменная механика!)**: по умолчанию круглый
  «постер» (фото + тонировка цветом категории + 6 мини-фигур, выступающих за
  круг); при ховере переворот в **цветной квадрат того же цвета** с минимальным
  контентом: тег типа (лаймовый), заголовок, клиент, фоновое текст-эхо ~5%.
  Никаких тегов-годов-описаний-CTA сверх этого. Реализовано в
  `src/components/CaseCard.tsx` + CSS в `index.css` (.case-card).

## Данные

- `src/data/pages/*.json` — контент каждой страницы, извлечённый с Tilda:
  блоки → тексты (с ролями title/subtitle/text), картинки, видео, в исходном
  порядке. Это источник правды для кейсов и страниц услуг.
- `src/data/cases.json` — реестр 43 кейсов: slug, route, категория, клиент,
  заголовок, обложка.
- `src/data/site.ts` — контакты, меню, услуги, факты, команда.
- `src/data/clients.ts` — 64 логотипа клиентов.
- `src/data/assets-manifest.json` — карта «URL на tildacdn → локальный путь».

### Ассеты

Картинки скачиваются одной командой:

```bash
npm run assets
```

Скрипт качает все 358 файлов в `public/assets/` и создаёт `.env.local` с
`VITE_LOCAL_ASSETS=1` — после этого сайт использует локальные файлы и
полностью независим от Тильды. До запуска скрипта сайт работает на
remote-URL (tildacdn), так что разработку можно начинать сразу.

## Роутинг

URL-структура сохранена 1:1 со старым сайтом (важно для SEO):
`/event/samsung`, `/video/patriot`, `/samara_vdnh` и т.д. Кейсы рендерятся
шаблоном `CasePage` через catch-all роут; услуги — `ServicePage`.

## Что доделать (по приоритету)

1. **Формы**: заполнить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в
   `public/api/lead.php`. Бот создаётся через @BotFather; chat_id — через
   @userinfobot. PHP-обработчик рассчитан на shared-хостинг
   (timeweb/beget/sprinthost). Альтернатива — Resend, как на weshow.su, но на
   shared-хостинге PHP `mail()` + Telegram надёжнее и бесплатно.
2. **Сверить фото команды с именами**: соответствие файлов
   `mriyaresort_-01-0X.png` именам в `src/data/site.ts` (TEAM) выставлено по
   порядку DOM и может не совпадать. Открыть старый сайт, сверить визуально.
3. **Showreel**: сейчас грузится с Dropbox (`CONTACTS.showreel`). Скачать
   `HM_Showreel.mp4`, положить в `public/video/` и поменять ссылку — иначе
   зависимость от Dropbox останется. То же для видео в кейсе `samara_vdnh`
   (6 dropbox-роликов) — либо перенести на VK Video и заменить на embed.
4. **Типографика кейсов**: `BlockView` рендерит контент «честно», но
   универсально. Для флагманских кейсов (samara_vdnh в первую очередь) стоит
   сделать индивидуальную вёрстку поверх данных из JSON.
5. **Заголовки/обложки кейсов**: просмотреть `/project` глазами — заголовки
   взяты из H1 старых страниц, часть можно отполировать в `cases.json`.
6. **Hero-логотип** (`/brand/logo_hero.svg`, 256 КБ, содержит растр) — при
   желании заменить на лёгкий чисто-векторный вариант из дизайн-системы v2.2.

## Запуск на хостинге (timeweb / beget / sprinthost / reg.ru)

```bash
npm install
npm run assets     # один раз: скачать графику с tildacdn
npm run sitemap    # обновить public/sitemap.xml
npm run build      # сборка в dist/
```

Залить на хостинг **содержимое `dist/`** в корень сайта (`public_html`).
В dist попадает всё из `public/`: ассеты, `api/lead.php`, `.htaccess`
(SPA-роутинг для Apache уже настроен), `robots.txt`, `sitemap.xml`.
Требование к хостингу: PHP ≥ 7.4 с curl (есть у всех перечисленных).

После переключения DNS:
- Проверить формы (отправка → /sk, заявка в Telegram).
- Яндекс Вебмастер: переобход + отправить sitemap.
- Метрика уже подключена (ID 71125393 в `index.html` — сверить с актуальным).
